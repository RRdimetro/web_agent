Диаграмма последовательности вызовов
📌 Сценарий работы агента:
1. Инициализация - загрузка config/agent_config.json, создание директорий, инициализация логгера.
2. Регистрация - POST /wa_reg/ с UID, получение access_code с автоматическими повторными попытками.
3. Основной цикл - периодический опрос сервера (POST /wa_task/) на наличие задач.
4. Выполнение задачи - обработка типов: FILE, TASK, CONF, TIMEOUT.
5. Отправка результата - POST /wa_result/ с кодом возврата и файлами.
6. Завершение - корректное завершение всех потоков и очистка ресурсов.

📌 Фаза 1: Инициализация

Загрузка config/agent_config.json, создание директорий, инициализация логгера. Время: ~100ms

📌 Фаза 2: Регистрация

POST /wa_reg/ с UID, получение access_code с retry (3 попытки). Время: ~200-500ms

📌 Фаза 3: Опрос задач

POST /wa_task/ каждые N секунд (по умолчанию 5 сек). При отсутствии задач - code_responce:"0"

📌 Фаза 4: Выполнение

Обработка типов задач: FILE (копирование), TASK (команда), CONF (изменение конфига), TIMEOUT (интервал)

📌 Фаза 5: Отправка результата

POST /wa_result/ multipart/form-data с result_code, result JSON, файлами. Время: ~100-500ms

📌 Фаза 6: Завершение

Остановка worker_thread, очистка ресурсов, корректный exit. Время: ~50-200ms
