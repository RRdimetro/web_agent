# Спецификация Web Agent API

## Общие сведения

### Базовый URL
```
https://{server_url}/app/webagent1/api/
```

### Протокол
- **Транспорт:** HTTPS
- **Формат данных:** JSON
- **Кодировка:** UTF-8

### Заголовки
```
Content-Type: application/json
```

---

## 1. Регистрация агента

### URI
```
POST /wa_reg/
```

### Тело запроса

```json
{
    "UID": "agent-001",
    "descr": "web-agent"
}
```

### Параметры запроса

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| UID | string | Да | Уникальный идентификатор агента |
| descr | string | Нет | Описание агента (по умолчанию "web-agent") |

### Успешный ответ (200 OK)

```json
{
    "code_responce": "0",
    "msg": "Регистрация прошла успешно",
    "access_code": "594807-1ddb-36af-9616-d8ed2b9d"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| code_responce | string | "0" — успех |
| msg | string | Сообщение |
| access_code | string | Код доступа для последующих запросов |

### Ошибка: агент уже зарегистрирован

```json
{
    "code_responce": "-3",
    "msg": "Такой агент уже зарегистрирован"
}
```

---

## 2. Получение задачи

### URI
```
POST /wa_task/
```

### Тело запроса

```json
{
    "UID": "agent-001",
    "descr": "web-agent",
    "access_code": "594807-1ddb-36af-9616-d8ed2b9d"
}
```

### Параметры запроса

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| UID | string | Да | UID агента |
| descr | string | Нет | Описание агента |
| access_code | string | Да | Код доступа (получен при регистрации) |

### Успешный ответ с задачей (code_responce = "1")

```json
{
    "code_responce": "1",
    "task_code": "CONF",
    "options": "{\"key\":\"poll_interval_sec\",\"value\":\"30\"}",
    "session_id": "bvLeD2gv-gtKH-IhmW-rsfd-Ejn1kyweawwi",
    "status": "RUN"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| code_responce | string | "1" — задача получена |
| task_code | string | Тип задачи: FILE, TASK, CONF, TIMEOUT |
| options | string | JSON с параметрами задачи |
| session_id | string | Идентификатор сессии задачи |
| status | string | Статус задачи |

### Успешный ответ без задачи (code_responce = "0")

```json
{
    "code_responce": "0",
    "status": "WAIT"
}
```

### Ошибка: неверный access_code

```json
{
    "code_responce": "-2",
    "msg": "неверный код доступа"
}
```

## 3. Типы задач

### 3.1 FILE — Найти и отправить файл

**options:**
```json
{"filename": "config.json"}
```

**Действие агента:**
1. Найти файл в рабочей директории
2. Скопировать в results_folder
3. Отправить на сервер через wa_result/

---

### 3.2 TASK — Выполнить команду

**options:**
```json
{"command": "uname -a"}
```

**Действие агента:**
1. Выполнить команду в shell
2. Зафиксировать код возврата и вывод
3. Отправить результат на сервер

---

### 3.3 CONF — Изменить параметр конфигурации

**options:**
```json
{"key": "poll_interval_sec", "value": "30"}
```

**Поддерживаемые ключи:**
- `poll_interval_sec` — интервал опроса (сек)
- `task_timeout` — таймаут выполнения задачи (сек)
- `max_poll_interval` — макс. интервал при ошибках (сек)
- `max_retries` — макс. попыток при ошибке
- `retry_delay` — задержка между попытками (сек)

**Действие агента:**
1. Обновить значение в конфиге
2. Применить изменение
3. Отправить статус на сервер

---

### 3.4 TIMEOUT — Изменить интервал опроса

**options:**
```json
{"interval": "15"}
```

**Действие агента:**
1. Установить новый poll_interval
2. Отправить статус на сервер

---

## 4. Отправка результата

### URI
```
POST /wa_result/
```

### Формат запроса
```
Content-Type: multipart/form-data
```

### Поля запроса

| Поле | Тип | Описание |
|------|-----|----------|
| result_code | int | Код результата: 0 — ok, < 0 — ошибка |
| result | string | JSON строка с деталями |
| file1, file2, ... | file | Файлы результата (опционально) |

### Формат поля result

```json
{
    "UID": "agent-001",
    "access_code": "594807-1ddb-36af-9616-d8ed2b9d",
    "message": "задание выполнено",
    "files": 3,
    "session_id": "ieL0LGzL-nyGP-mfG5-m3nI-eYL1CZzcaXz0"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| UID | string | UID агента |
| access_code | string | Код доступа |
| message | string | Сообщение о результате |
| files | int | Количество прикреплённых файлов |
| session_id | string | ID сессии задачи |

### Успешный ответ (200 OK)

```json
{
    "code_responce": "0",
    "msg": "ok"
}
```

### Ошибка

```json
{
    "code_responce": "-3",
    "msg": "не все файлы загружены",
    "status": "ERROR"
}
```

---

## 5. Коды ответов

| Код | Название | Описание | Действие агента |
|-----|----------|----------|-----------------|
| `0` | OK | Успех | Продолжить работу |
| `-2` | INVALID_ACCESS_CODE | Неверный access_code | Очистить код, повторная регистрация |
| `-3` | ALREADY_EXISTS / UPLOAD_ERROR | Агент существует / Ошибка загрузки | Логирование |
| `-13` | INVALID_ACCESS_CODE | Неверный код доступа | Очистить код, повторная регистрация |

## 6. Примеры сценариев

### Сценарий 1: Регистрация и выполнение задачи

**Шаг 1: Регистрация**
```
POST /wa_reg/
{"UID": "007", "descr": "web-agent"}

Ответ:
{"code_responce": "0", "access_code": "e87ccd-3146-0dcc-2aeb-796c4724"}
```

**Шаг 2: Получение задачи**
```
POST /wa_task/
{"UID": "007", "descr": "web-agent", "access_code": "e87ccd-3146-0dcc-2aeb-796c4724"}

Ответ:
{"code_responce": "1", "task_code": "TASK", "options": "{\"command\":\"uname -a\"}", "session_id": "..."}
```

**Шаг 3: Выполнение команды**
```bash
uname -a
# Linux hostname 6.17.0...
```

**Шаг 4: Отправка результата**
```
POST /wa_result/
Content-Type: multipart/form-data

result_code: 0
result: {"UID":"007","access_code":"...","message":"ok","files":0,"session_id":"..."}

Ответ:
{"code_responce": "0", "msg": "ok"}
```

---

### Сценарий 2: Изменение конфигурации

**Задача CONF:**
```json
{
    "code_responce": "1",
    "task_code": "CONF",
    "options": "{\"key\":\"poll_interval_sec\",\"value\":\"30\"}",
    "session_id": "..."
}
```

**Действие агента:**
1. Установить poll_interval = 30 сек
2. Отправить результат:
   ```
   result_code: 0
   result: {"message": "poll_interval_sec = 30", ...}
   ```

---

### Сценарий 3: Ошибка доступа

**Запрос задачи с неверным кодом:**
```
POST /wa_task/
{"UID": "007", "access_code": "invalid-code"}

Ответ:
{"code_responce": "-2", "msg": "неверный код доступа"}
```

**Действие агента:**
1. Очистить access_code
2. Выполнить повторную регистрацию
3. Получить новый access_code

---

## 7. Временные параметры

| Параметр | Значение |
|----------|----------|
| Таймаут соединения | 30 сек |
| Таймаут запроса | 60 сек |
| Интервал опроса | 2-300 сек (настраивается) |
| Макс. попыток | 3 |
| Задержка при ошибке | экспоненциальная |

---

## 8. Безопасность

- Все запросы по HTTPS
- Проверка SSL сертификата
- access_code не хранится в логах
- Валидация UID и параметров
