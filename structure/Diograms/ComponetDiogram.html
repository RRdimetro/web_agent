Диаграмма компонентов агента
📌 Описание архитектуры:
Агент состоит из нескольких независимых модулей, каждый из которых отвечает за свою функциональность.

🔷 Application Layer: Главный контроллер и конечный автомат, управляющие жизненным циклом.
🔶 Core Services: Конфигурация, логирование, безопасность.
🔷 Network Layer: HTTP клиент и API клиент для взаимодействия с сервером.
🔶 Execution Layer: Пул потоков и исполнители задач (процессы и команды).
🔷 File System: Работа с файлами и директориями.
🔶 External Systems: Внешние системы: сервер, ОС, файловая система.

📡 Протокол взаимодействия: HTTPS с форматом данных JSON.

📱 Application Layer

Application - главный контроллер, управляет жизненным циклом, обрабатывает сигналы.
StateMachine - конечный автомат, управляет состояниями агента.

⚙️ Core Services

Config - загрузка и валидация JSON конфигурации.
Logger - потокобезопасное логирование (spdlog).
SecurityManager - валидация команд и путей, белый список.

🌐 Network Layer

ApiClient - формирование запросов, парсинг ответов.
HttpClient - выполнение HTTPS запросов с retry (CPR/libcurl).

⚡ Execution Layer

TaskRunner - пул потоков, очередь задач.
ProcessExecutor - запуск процессов (Boost.Process).
CommandExecutor - выполнение системных команд.

💾 File System Layer

FileManager - операции с файлами и директориями (std::filesystem).
Создание, чтение, запись, перемещение, удаление.

🖧 External Systems

Server - удаленный сервер управления.
OperatingSystem - запуск процессов, выполнение команд.
FileSystem - хранение файлов задач, результатов, логов.
