Диаграмма последовательности вызовов
📌 Сценарий работы агента:
1. Инициализация - загрузка конфигурации, создание директорий, инициализация логгера.
2. Регистрация - отправка UID на сервер, получение sessionId с автоматическими повторными попытками.
3. Основной цикл - периодический опрос сервера на наличие задач.
4. Выполнение задачи - запуск процесса или команды в отдельном потоке.
5. Отправка результата - передача кода возврата и вывода на сервер.
6. Завершение - корректное завершение всех потоков и очистка ресурсов.

📌 Фаза 1: Инициализация

Загрузка config.json, создание директорий, инициализация логгера. Время: ~100ms

📌 Фаза 2: Регистрация

POST /register с retry механизмом (3 попытки). Получение sessionId. Время: ~200-500ms

📌 Фаза 3: Опрос задач

GET /task каждые N секунд (по умолчанию 10 сек). При отсутствии задач - 204 No Content

📌 Фаза 4: Выполнение

Запуск процесса/команды в отдельном потоке с таймаутом. Время: зависит от задачи

📌 Фаза 5: Отправка результата

POST /result с retry. Передача exit code, stdout, stderr. Время: ~100-500ms

📌 Фаза 6: Завершение

Остановка потоков, очистка ресурсов, корректный exit. Время: ~50-200ms
