
**Описание:** Диаграмма показывает взаимодействие компонентов агента с сервером и внешними системами.

**Потоки данных:**
1. Конфигурация → Configuration Module → остальные модули
2. Регистрация → Application → ApiClient → HttpClient → Server
3. Задача → Server → HttpClient → ApiClient → TaskRunner → Executor
4. Результат → Executor → TaskRunner → ApiClient → HttpClient → Server
5. Логи → Все модули → Logger → agent.log

---

## 5. Модульная структура

### 5.1 Модуль конфигурации
**Назначение:** Загрузка, парсинг, валидация и предоставление доступа к настройкам.

**Формат:** JSON

**Основные методы:**
- `loadFromFile()` - загрузка конфигурации
- `validate()` - валидация параметров
- Геттеры для всех параметров

### 5.2 HTTP-модуль
**Назначение:** Выполнение HTTPS запросов с retry и таймаутами.

**Библиотека:** CPR (обертка над libcurl)

**Особенности:**
- Автоматический retry при сетевых ошибках
- Экспоненциальная задержка между попытками
- Поддержка SSL/TLS
- Таймауты соединения и запроса

### 5.3 Модуль выполнения задач
**Назначение:** Запуск процессов и команд, сбор результатов.

**Типы задач:**
- PROCESS_EXECUTION - запуск исполняемого файла
- COMMAND_EXECUTION - выполнение системной команды
- FILE_UPLOAD - передача файла на сервер
- FILE_DOWNLOAD - скачивание файла с сервера

**Особенности:**
- Поддержка таймаутов выполнения
- Возможность отмены задачи
- Захват stdout и stderr
- Получение кода возврата

### 5.4 Модуль файловой системы
**Назначение:** Операции с файлами и директориями.

**Функции:**
- Создание и удаление директорий
- Чтение и запись файлов
- Перемещение и копирование
- Поиск файлов по шаблону
- Создание временных директорий

### 5.5 Модуль логирования
**Назначение:** Потокобезопасное логирование с ротацией файлов.

**Библиотека:** spdlog

**Уровни:** TRACE, DEBUG, INFO, WARNING, ERROR, CRITICAL

**Особенности:**
- Потокобезопасность
- Ротация файлов по размеру
- Форматированный вывод
- Вывод в файл и консоль

## 6. Обработка ошибок и восстановление

### 6.1 Сетевые ошибки
- Автоматический retry с экспоненциальной задержкой
- Максимум 3 попытки
- При истечении сессии - повторная регистрация

### 6.2 Ошибки выполнения задач
- Фиксация кода возврата
- Запись stdout и stderr
- Передача ошибки на сервер

### 6.3 Таймауты
- Таймаут соединения (30 сек)
- Таймаут запроса (60 сек)
- Таймаут выполнения задачи (300 сек)

### 6.4 Восстановление состояния
- При ошибке агент возвращается в состояние IDLE
- При фатальной ошибке - корректное завершение
- Возможность перезапуска из состояния ERROR

## 7. Безопасность

### 7.1 HTTPS
- Все запросы к серверу выполняются по HTTPS
- Проверка SSL сертификата
- Возможность указать путь к CA сертификатам

### 7.2 Валидация входных данных
- Белый список разрешенных команд
- Черный список запрещенных путей
- Санитизация пользовательского ввода
- Ограничение размера файлов

### 7.3 Безопасное хранение
- Отсутствие хранения чувствительных данных в открытом виде
- Конфигурация не содержит паролей
- Логи не содержат секретной информации

### 8.1 Требования

- Ленивая загрузка ресурсов

### 8.3 Мониторинг
- Логирование времени выполнения задач
- Сбор метрик (CPU, память)

## 10. Заключение

Разработанная архитектура WEB-AGENT обеспечивает:

| Требование | Реализация |
|------------|------------|
| **Кроссплатформенность** | Поддержка Windows, Linux, macOS |
| **Модульность** | Четкое разделение на независимые компоненты |
| **Многопоточность** | Пул потоков, потокобезопасное логирование |
| **Надежность** | Retry механизмы, обработка таймаутов, восстановление |
| **Безопасность** | HTTPS, валидация, белые/черные списки |
| **Производительность** | Минимальное потребление ресурсов в режиме ожидания |

Все компоненты спроектированы с учетом дальнейшего расширения и модификации. Архитектура готова к реализации на C++17 с использованием современных библиотек.
